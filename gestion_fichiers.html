Système de Gestion de Fichiers

Un véritable système de gestion de fichiers développé en C, implémentant les opérations sur les fichiers et la gestion des répertoires. Ce projet met en avant la manipulation bas-niveau, la gestion de la mémoire et la création d'une interface en ligne de commande pour simuler un mini système de fichiers.

Fonctionnalités principales

  • Création, suppression, renommage de fichiers et dossiers
  • Navigation dans l'arborescence des répertoires
  • Lecture et écriture dans les fichiers
  • Gestion des permissions
  • Interface en ligne de commande intuitive

Analyse du Code et Recommandations

1. Overview

Ce projet est un programme C qui implémente un système de gestion de la mémoire secondaire, simulant des opérations sur fichiers et l'allocation/libération mémoire.

2. Principales Structures de Données

  • Record : id, isDeleted (pour chaque enregistrement)
  • Metadata : infos sur chaque fichier (nom, nb blocs, organisation, etc.)
  • Block : tableau d'enregistrements, pointeur vers bloc suivant (chaînage), nom de fichier
  • Memoiresecondaire : structure principale, contient tous les blocs, la table d'allocation, les métadonnées, etc.

3. Fonctionnalités Clés

  • Initialisation mémoire, allocation dynamique
  • Création, suppression, renommage de fichiers
  • Insertion, recherche, suppression logique/physique d'enregistrements
  • Compactage et défragmentation mémoire

4. Points Forts

  • Gestion mémoire complète (contiguë/chaînée, triée/non triée)
  • Gestion des erreurs (doublons, mémoire, existence, etc.)
  • Interface utilisateur claire, messages colorés

5. Axes d'Amélioration

  • Vérification des bornes et sécurité mémoire
  • Optimisation des recherches (ex : dichotomie pour trié)
  • Organisation du code en plusieurs fichiers, documentation
  • Ajout de persistance, transactions, sauvegarde/restauration

6. Détails Techniques

#define MAX_NOMFICHIER 50
#define MAX_FILES 100
#define MAX_RECORDS 1000
  • Organisation globale : 'C' (contiguë), 'L' (chaînée)
  • Organisation interne : 'T' (triée), 'N' (non triée)

7. Recommandations

  • Séparer le code en plusieurs fichiers (header, implémentation, main)
  • Ajouter des codes d'erreur, logs, tests unitaires
  • Documenter chaque fonction, ajouter un manuel utilisateur
  • Ajouter validation des entrées, sécurité, options de chiffrement

8. Conclusion

Le code offre une base solide pour un gestionnaire mémoire. Avec les améliorations proposées, il peut devenir un système robuste et prêt pour la production.
